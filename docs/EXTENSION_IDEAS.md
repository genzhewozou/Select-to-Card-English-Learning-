# 英语学习系统 - 扩展功能与优化建议

基于当前代码与业务，从「复习体验、数据统计、文档与卡片联动、效率与安全」等方向整理的扩展点与优先级建议。

---

## 一、当前已有能力简要

| 模块 | 已有功能 |
|------|----------|
| 用户 | 注册、登录（localStorage）、设置页 AI 配置 |
| 文档 | 上传 Word/TXT、列表、查看正文、删除 |
| 文档内 | 选中文本 → 生成卡片、开启 AI 时弹窗内自动生成背面释义/例句并可直接编辑 |
| 卡片 | 创建（来自文档/手动）、列表、编辑、删除；正面/背面、注释、进度 |
| 复习 | 今日待复习列表、正面→翻面→熟练度 1–5→艾宾浩斯更新下次复习 |
| 进度 | 熟练度、复习次数、下次复习时间（EbbinghausUtil） |

---

## 二、高价值、易落地的扩展（优先考虑）

### 1. 复习页体验增强

- **复习时展示注释**  
  当前只展示正面/背面，可在「显示背面」后同时展示该卡片的 `notes`（AI 注释），方便回忆和巩固。
- **键盘快捷键**  
  - 空格：翻面  
  - 数字 1–5：直接选熟练度并提交下一张  
  减少鼠标操作，适合连续刷题。
- **复习进度条**  
  顶部显示「第 3 / 20 张」并带进度条，完成一张即更新，增强完成感。

**实现要点**：前端 Review 页拉取卡片时带上 `notes`（若接口未带可在 `getCard`/列表接口中保证返回），渲染背面时一并展示；键盘事件 `onKeyDown` 绑定空格与 1–5。

---

### 2. 学习统计 / 简单仪表盘

- **首页或独立「统计」页**  
  - 今日：待复习数量、已完成复习数量  
  - 汇总：卡片总数、来自文档的卡片数、总复习次数  
  - 可选：本周/本月「每日复习数」折线图（需按日聚合）
- **后端**  
  - 新增统计接口，例如：`GET /api/stats/overview` 返回上述汇总与今日数据；若做图表，再增加 `GET /api/stats/review-history?from=&to=` 按日返回复习次数。

**价值**：让用户看到学习进度和连续性，提升留存与动力。

---

### 3. 文档内高亮与跳转（利用现有 documentId/offset）

- 卡片实体已有 `documentId`、`startOffset`、`endOffset`，但文档查看页尚未使用。
- **在文档正文中**  
  - 对「已存在卡片」的选段做高亮（例如浅色底或下划线）。  
  - 点击高亮区域可弹出：查看对应卡片、编辑、或「加入今日复习」。
- **从卡片跳回原文**  
  - 卡片详情/编辑页若存在 `documentId`，提供「在文档中定位」按钮，跳转到文档页并滚动到对应位置（用 offset 或前端解析段落）。

**实现要点**：文档列表/详情接口返回卡片列表或段落级「是否有卡片」信息；或由前端在进入文档页时请求该文档下的卡片列表，再根据 offset 在正文中渲染高亮与点击区域。

---

### 4. 卡片列表筛选与搜索

- **按文档筛选**  
  列表接口已支持 `documentId`，前端增加「按文档」下拉筛选即可。
- **关键词搜索**  
  后端 `GET /api/card/list?keyword=xxx` 对正面/背面（及可选注释）做模糊查询；前端搜索框绑定该参数。
- **按状态筛选**  
  - 仅「今日待复习」  
  - 按熟练度（如只显示 1–2 级）  
  - 按「下次复习时间」区间  

**价值**：卡片多了之后便于管理和针对性复习。

---

### 5. 错题本 / 薄弱卡片

- **概念**：将「熟练度 1–2」或「最近一次评为不熟」的卡片视为错题/薄弱项。
- **实现**：  
  - 后端：`GET /api/review/weak` 或 `GET /api/card/list?proficiencyMax=2` 返回这些卡片。  
  - 前端：复习页或导航中增加「错题本」入口，列表与复习流程复用现有组件，仅数据源不同。

**价值**：集中巩固不熟内容，符合学习习惯。

---

## 三、中期可做的功能

### 6. 批量操作

- **批量删除**  
  卡片列表支持多选 + 「批量删除」按钮，后端 `POST /api/card/batch-delete` 接收 id 列表。
- **按文档批量生成卡片**  
  - 例如：上传生词表（每行一个词）或从文档中解析出「未生成卡片的生词」，一键批量创建卡片（可勾选是否调用 AI 生成背面）。  
  - 需考虑限流与任务化（大批量可做异步任务 + 进度查询）。

---

### 7. 认证与安全增强

- **JWT 或 Session**  
  替代当前仅靠 `X-User-Id` 的方式，登录后下发 token，请求头带 `Authorization`，后端校验。
- **密码**  
  存储前 BCrypt 等哈希；提供「修改密码」入口。
- **设置页**  
  可增加「修改密码」「退出登录」等。

---

### 8. AI 相关扩展

- **发音（TTS）**  
  正面/背面旁增加「朗读」按钮，调用浏览器 TTS 或第三方 TTS API（需考虑配额与隐私）。
- **一键出题**  
  用 AI 根据背面/注释生成填空、选择等题目，存为「题目」或直接作为卡片的另一种展示形式（可后续再设计数据模型）。
- **同义词/词根**  
  生成卡片时或编辑时，可选调用 AI 补充「同义词」「词根词缀」等字段并展示在复习或列表中。

---

### 9. 导入导出

- **导出**  
  将当前用户卡片导出为 CSV/Anki 格式，便于备份或在 Anki 中使用。
- **导入**  
  支持 CSV/Excel 批量导入（正面、背面、可选标签），自动创建卡片并可勾选是否调用 AI 补全背面。

---

### 10. 标签与分类

- **卡片打标签**  
  如「CET-6」「雅思」「易错」等；需新增 tag 表或 card_tag 关联表。
- **按标签筛选与复习**  
  列表和复习页支持「按标签」筛选，或「只复习某标签」的复习模式。

---

## 四、体验与产品级优化

- **首页**  
  未登录跳登录；已登录可跳转到「今日复习」或「统计概览」，而不是空白或直接文档列表。
- **复习提醒**  
  浏览器 Notification API 在「今日有待复习且用户多日未打开」时提醒（需简单记录上次访问/上次复习时间）。
- **移动端 / PWA**  
  响应式布局、触摸友好的复习操作；PWA 可安装到桌面，离线缓存静态资源与部分数据。
- **暗色模式**  
  设置中增加主题切换，CSS 变量或 Ant Design 主题切换即可。

---

## 五、建议落地顺序（按优先级）

1. **复习页增强**（展示注释 + 快捷键 + 进度条）—— 直接提升核心复习体验。  
2. **学习统计 / 仪表盘**（今日 + 汇总 + 可选简单图表）—— 提升成就感和留存。  
3. **文档内高亮与跳回原文**—— 用好现有 documentId/offset，强化「文档→卡片」闭环。  
4. **卡片筛选与搜索**—— 数据量上来后必做。  
5. **错题本**—— 逻辑简单，复用现有复习流程。  
6. 再按需做：批量操作、认证加固、AI 扩展、导入导出、标签等。

---

## 六、小结

当前系统已经具备「文档 → 选词生成卡片 → AI 注释 → 艾宾浩斯复习」的完整链路；扩展上优先把**复习体验**和**数据可见性**（统计）做好，再围绕**文档与卡片联动**（高亮、跳转）和**筛选/错题本**深化，最后再考虑安全、批量、AI 与导入导出等。以上功能均可按模块分步实现，与现有技术栈兼容。
